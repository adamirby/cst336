A relational database is built from a collection of related tables. Each table consists of a collection of rows and columns. Each column pertains to a field of data, and each row represents a record in the table. Each table has a collection of 'keys' or fields that each record must have a unique value. There is a 'primary key' that is the main unique identifier for the records in the table. The many tables in a database are related to other tables through the use of 'foreign keys' which are fields that directly correlate with the 'keys' in other tables that define the relational nature of the database.
